================================================================================
รายงานรายละเอียดระบบเช็คชื่อเข้าฝึก นศท. ด้วยใบหน้าและตำแหน่งที่ตั้ง
(Face Recognition & Geofence Attendance System for ROTC / นศท.)
ศูนย์ฝึกย่อยโรงเรียนสมเด็จพิทยาคม
================================================================================

1. ภาพรวมโปรเจกต์ (PROJECT OVERVIEW)
--------------------------------------------------------------------------------
ระบบเช็คชื่อเข้าฝึกนักศึกษาวิชาทหาร (นศท. Check) เป็นระบบเว็บแอปพลิเคชันอัจฉริยะที่พัฒนาขึ้นเพื่อเพิ่มประสิทธิภาพและความแม่นยำในการเช็คชื่อเข้าร่วมการฝึกของนักศึกษาวิชาทหาร (นศท.) ในแต่ละวันฝึก ช่วยป้องกันการเช็คชื่อแทนกัน อำนวยความสะดวกในการยื่นและพิจารณาอนุมัติใบลาผ่านระบบออนไลน์ พร้อมระบบแจ้งเตือนทางอีเมลอัตโนมัติ และระบบรายงานสรุปยอดส่งตรงไปยังอาจารย์ผู้รับผิดชอบของแต่ละสถานศึกษา

2. วัตถุประสงค์ของระบบ (SYSTEM OBJECTIVES)
--------------------------------------------------------------------------------
1. เพิ่มความแม่นยำและป้องกันการเช็คชื่อแทนกัน ด้วยเทคโนโลยีจดจำใบหน้า (Face Recognition AI) ร่วมกับการตรวจสอบพิกัดภูมิศาสตร์ (Geofence GPS)
2. อำนวยความสะดวกให้ นศท. ในการยื่นคำร้องขอลากิจ/ลาป่วยออนไลน์ พร้อมแนบหลักฐานการลา
3. ลดขั้นตอนการทำงานของครูฝึกด้วยระบบพิจารณาอนุมัติใบลาออนไลน์ และส่งผลการอนุมัติแจ้งกลับไปยังอีเมล นศท. ทันที
4. เพิ่มความสะดวกในการสลับบัญชีผู้ใช้ (Multi-Account Switcher) สำหรับ นศท. ที่ใช้อุปกรณ์ร่วมกัน
5. สร้างรายงานสรุปยอดการเช็คชื่อเข้าฝึกประจำวัน (มาเรียน, มาสาย, ขาดเรียน, ลา) และส่งรายงานเข้าอีเมลสถานศึกษาโดยอัตโนมัติ

3. เทคโนโลยีที่ใช้ในการพัฒนา (TECHNOLOGY STACK & ARCHITECTURE)
--------------------------------------------------------------------------------
- Frontend Logic: JavaScript (ES6+ Vanilla JavaScript)
- HTML & Styling: HTML5, Vanilla CSS, TailwindCSS Design System (ฟอนต์ IBM Plex Sans Thai, Prompt, Kanit)
- Database & Storage: Firebase Firestore, Firebase Storage
- Authentication: Firebase Authentication (Email/Password Auth)
- AI & Face Recognition: face-api.js (Tiny Face Detector, Face Landmark 68, Face Recognition Weights)
- Backend Email Automation: Google Apps Script (GAS) Web App + GmailApp API
- Deployment Platform: Vercel Cloud Platform / Web Server

4. โครงสร้างระดับสิทธิ์การใช้งาน (USER ROLES & PERMISSIONS HIERARCHY)
--------------------------------------------------------------------------------
- ระดับ 1: นักศึกษาวิชาทหาร (Student - Role 1)
  * เช็คชื่อเข้าเรียน/เลิกเรียน ด้วยการสแกนใบหน้าและตรวจสอบตำแหน่ง GPS
  * ส่งคำร้องขอลากิจ / ลาป่วย พร้อมแนบรูปภาพหลักฐาน และกรอกอีเมลรับแจ้งผล
  * ดูประวัติการเช็คชื่อและประวัติการขอลาของตนเอง
  * สลับบัญชีใช้งานกรณีใช้อุปกรณ์ร่วมกับเพื่อน นศท.

- ระดับ 2: หัวหน้าหมวด (Platoon Leader - Role 2)
  * ดูสรุปยอดและรายชื่อ นศท. เฉพาะในหมวดปีการฝึกตนเอง
  * กรองข้อมูลแยกตามเพจและสายการฝึก (หมวดปกติ ชาย / หมวดพิเศษ หญิง) อย่างเด็ดขาด

- ระดับ 3: ผู้กุมกองร้อย (Company Commander - Role 3)
  * ดูสรุปยอดและรายชื่อ นศท. ในกองร้อย/ชั้นปีการฝึกตนเอง
  * ดูรายงานภาพรวมของกองร้อยตามสายการฝึก (ปกติ/พิเศษ)

- ระดับ 4: กองพัน / ครูฝึก (Battalion / Instructor - Role 4)
  * ดูสรุปยอดและสถิติภาพรวมการฝึกของ นศท. ทุกหมวดและทุกโรงเรียน (Read-only)

- ระดับ 5: หัวหน้าชุดครูฝึก / ผู้อนุมัติใบลา (Chief Instructor - Role 5)
  * พิจารณา อนุมัติ / ปฏิเสธ คำร้องขอลาของ นศท. พร้อมส่งอีเมลแจ้งผลกลับอัตโนมัติ
  * จัดการข้อมูลสถานศึกษา อีเมลอาจารย์ผู้รับผิดชอบ และกดส่งรายงานสรุปเข้าอีเมลทุกโรงเรียน

- ระดับ 6: ผู้ดูแลระบบ (System Admin - Role 6)
  * สิทธิ์สูงสุดในการเข้าถึงระบบ จัดการเพิ่ม/แก้ไข/ลบบัญชี นศท. และเจ้าหน้าที่
  * ลงทะเบียนสแกนใบหน้า (Face ID) ให้กับ นศท.
  * กำหนดพิกัดและรัศมีเขตเช็คชื่อ (Geofence Setup) และตั้งค่ากฎเวลาเช็คชื่อ
  * ตั้งค่า Web App URL ของ Google Apps Script และอีเมลผู้อนุมัติหลัก

5. ฟังก์ชันการทำงานหลัก (CORE SYSTEM FEATURES)
--------------------------------------------------------------------------------
1. ระบบเช็คชื่อด้วยใบหน้าและพิกัด (Face & Geofence Check-in)
   - ตรวจสอบว่าตำแหน่งของ นศท. อยู่ภายในรัศมีที่กำหนดหรือไม่ (เช่น 200 เมตร จากศูนย์ฝึก)
   - ตรวจสอบใบหน้าจริงเปรียบเทียบกับ Face Vector (128 Dimensions) ที่ลงทะเบียนไว้
   - บันทึกสถานะ "มาเรียน" หรือ "มาสาย" อัตโนมัติตามช่วงเวลาที่กำหนด

2. ระบบเช็คยอดระหว่างวัน / กลางวัน (Midday Attendance Tracking)
   - ครูฝึกหรือหัวหน้าหมวดสามารถเปิดรอบเช็คยอดระหว่างวันได้
   - บันทึกสถานะ นศท. (คงยอด, ปฏิบัติภารกิจ, ไม่อยู่/ขาด) พร้อมถ่ายภาพหรือสแกนใบหน้ายืนยัน

3. ระบบการลาและส่งอีเมลแจ้งเตือนอัตโนมัติ (Online Leave Management & Email Automation)
   - นศท. กรอกข้อมูลการลา เลือกประเภท วันที่ เหตุผล แนบหลักฐาน พร้อมระบุอีเมล/เบอร์โทรติดต่อ
   - ระบบยิง Webhook ไปยัง Google Apps Script เพื่อส่งอีเมลแจ้งเตือนครูฝึกผู้อนุมัติทันที
   - เมื่อผู้อนุมัติทำรายการ อนุมัติ/ปฏิเสธ ระบบจะส่งอีเมลผลการอนุมัติแจ้งกลับไปยัง นศท. ทันที

4. ระบบสลับบัญชีผู้ใช้งาน (Multi-Account Switcher)
   - จดจำบัญชีผู้ใช้ในเบราว์เซอร์ ช่วยให้สลับเปลี่ยนสลับบัญชีได้รวดเร็วเพียง 1 คลิก
   - มีระบบ Switch Guard ป้องกันหน้าเว็บดีดหลุดขณะสลับบัญชี

5. ระบบรายงานและส่งอีเมลสรุปถึงสถานศึกษา (School Summary Daily Reports)
   - สรุปยอด นศท. แยกตามโรงเรียน (มาเรียน, มาสาย, ขาดเรียน, ลา)
   - ส่งอีเมลรายงานสรุปประจำวันถึงอาจารย์ผู้รับผิดชอบของแต่ละโรงเรียน
   - กรองซ่อนรายชื่อคนมาเรียน เพื่อแสดงเฉพาะรายชื่อ นศท. ที่ ขาด / สาย / ลา ช่วยให้อีเมลกระชับ อ่านง่าย

6. โครงสร้างฐานข้อมูล (FIRESTORE DATABASE SCHEMA)
--------------------------------------------------------------------------------
1. Collection: `users`
   - Document ID: Auth UID
   - Fields: studentId, name, role (1-6), company (ชั้นปี), platoon (หมวด), school, track (regular/special), contactEmail, phone, faceDescriptor, active

2. Collection: `attendance`
   - Path: `attendance/{dateStr}/records/{uid}`
   - Fields: uid, studentId, name, company, platoon, school, status (present/late/leave/absent), checkInTime, checkOutTime, faceVerified, location

3. Collection: `leave_requests`
   - Document ID: Auto ID
   - Fields: id, uid, studentId, studentName, studentEmail, studentPhone, company, platoon, school, type (personal/sick), startDate, endDate, days, reason, evidenceUrl, status (pending/approved/rejected), approvedBy, approvedAt, approverNote

4. Collection: `schools`
   - Document ID: Auto ID
   - Fields: name, code, teacherName, teacherEmail, teacherPhone, center

5. Collection: `settings`
   - Document `geofence`: center (lat, lng), radius (meters)
   - Document `gas`: url (Google Apps Script Web App URL), approverEmail, adminEmail
   - Document `attendance-rules`: checkInStart, checkInEnd, checkOutStart, checkOutEnd, enforceTimeWindow, systemEnabled

7. สรุปคุณค่าและความสำเร็จของระบบ (SUMMARY & KEY ACHIEVEMENTS)
--------------------------------------------------------------------------------
- ป้องกันการทุจริตเช็คชื่อแทนกันได้ 100% ด้วยการผสานปัญญาประดิษฐ์ตรวจจับใบหน้าและพิกัดดาวเทียม
- ลดภาระงานเอกสารและกระดาษใบลา เปลี่ยนเป็นกระบวนการดิจิทัลเต็มรูปแบบ
- มีระบบการสื่อสารผ่านอีเมลอัตโนมัติ เชื่อมโยงระหว่าง นศท., ครูฝึก และอาจารย์ผู้ประสานงานโรงเรียน
- รองรับการใช้งานอย่างยืดหยุ่นทั้งบนสมาร์ตโฟน (Mobile Responsive) และคอมพิวเตอร์ (Desktop Web)
